# 인터페이스별 페르소나 템플릿

## Web UI 페르소나 예시

| ID      | 이름         | 역할                 | 업무 목표                         | 오늘의 시나리오                  | 핵심 검증                   |
| ------- | ------------ | -------------------- | --------------------------------- | -------------------------------- | --------------------------- |
| P-WEB-1 | 신규 사용자  | 첫 방문자            | 회원가입 후 핵심 기능 하나를 완료 | 가입→온보딩→첫 작업→완료         | 온보딩 흐름, 가치 전달 속도 |

## 크로스 인터페이스 시나리오 템플릿

### CROSS-1: "한 인터페이스에서 생성, 다른 인터페이스에서 확인"

| 단계 | 인터페이스 | 행동             | 검증             |
| ---- | ---------- | ---------------- | ---------------- |
| 1    | A          | 데이터 생성      | 생성 성공        |
| 2    | B          | 동일 데이터 조회 | 즉시 반영되는가? |
| 3    | B          | 데이터 수정      | 수정 성공        |
| 4    | A          | 수정 결과 확인   | 동기화 확인      |

### CROSS-2: "풀사이클을 여러 인터페이스로"

| 단계 | 인터페이스 | 행동           | 검증            |
| ---- | ---------- | -------------- | --------------- |
| 1    | A          | 작업 시작      | 생성            |
| 2    | B          | 상태 확인      | A의 변경이 반영 |
| 3    | C          | 코멘트/수정    | 업데이트        |
| 4    | A + B      | 최종 상태 확인 | 전체 동기화     |
